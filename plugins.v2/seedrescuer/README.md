# 种子找回助手 (SeedRescuer) v6.1.2

本插件专为 MoviePilot V2 设计，用于极速扫描本地存储目录，精准提取丢失种子的“孤儿资源”。支持**纯本地扫描导出**，或配置 **Prowlarr 直连引擎** 实现从检索到【内存直写】底层下载器的全自动化辅种闭环。

## ⚠️ 极度重要与免责声明 (使用前必读)

🚨 **【保命操作：手动强制校验】**
全量找回任务完成后，虽然插件默认以“暂停”状态将种子安全推送到下载器，但**强烈建议您在下载器 (qBittorrent / Transmission) 中全选这些新加入的种子，手动执行一次“强制校验 (Force Recheck)”！** 请务必确认校验进度条跑满 **100%** 后，再点击“开始”恢复做种。如果在校验过程中发现进度条停在 **0%** 并试图重新下载，请**立刻删除该任务**！这说明文件版本不符或路径映射配错，继续下载将会**不可逆地覆盖重写**您本地的原有文件，并可能白白耗尽您的 PT 站点下载配额！

📜 **【免责声明】**

## 🌟 核心特性 (v6.1.2 全新架构)

- **🚀 Prowlarr 协议直连引擎**：彻底剥离 MoviePilot 内置搜索链路。通过 HTTP API 直连 Prowlarr，获取最纯净的 Raw 数据，无视上层复杂的媒体识别过滤，真正做到“有种就能搜到”。
- **💉 内存级直写推送 (零干预)**：针对 TR/QB 进行智能降维打击。插件将提取真正的 `.torrent` 字节流至系统内存，强行喂给下载器底层接口。**绝对绕过任何后置整理或重命名逻辑**，确保本地文件 100% 完美校验。
- **🛡️ 严苛的防风控流控 (专治 503)**：内置全局随机休眠（推荐 45~90 秒）与变种词局部强制冷却（15 秒）双重限流机制。完美适配 M-Team 等严苛站点的 API 并发防御，挂机一整天也绝不关小黑屋。
- **⚖️ 极致匹配算法**：精准提取本地文件夹总大小，与 Prowlarr 返回源进行比对（容忍误差 ≤ 3%），并自动校验 `WEB-DL`, `REMUX`, `iQIYI` 等核心压制组特征，拒绝李鬼。
- **孤儿资源极速提取**：即便不配置 Prowlarr，本插件也是一个极致纯净的【孤儿种子扫描仪】。它能自动过滤已在下载器中做种的健康资源，提取出待辅种的绝对路径和纯净片名列表，供你手工一键复制去 PT 站补种。
- **🛑 紧急制动系统**：新增前端“强制停止任务”红色按钮，发现配置有误或路径重叠时，可一键紧急刹车，后台线程安全退出。

## 📖 使用指南

### 1. 基础配置
- **扫描根路径**：填写你的影视存放路径，多路径用逗号分隔（如 `/media/movies, /media/tv`）。支持多级目录深度钻取，遇到影视文件夹自动截断。
- **Prowlarr 检索源 (强烈推荐)**：填写 URL（如 `http://192.168.1.100:9696`）及 API Key。配置后即激活自动化找回引擎；留空则仅作为本地扫描导出工具。
- **防封延迟设置**：检索最小/最大延迟建议设置为 **45** 和 **90**，这是保护你 PT 账号安全的生命线。
- **目标下载器**：必选！用于比对跳过已有种子及接收推送。

### 2. 路径转换 (Docker 跨容器必填)
如果 MoviePilot 扫描到的路径与下载器看到的路径不一致，请务必配置 `路径转换`。
- 格式：`MoviePilot容器内路径:下载器容器内路径`
- 示例：`/media/movies:/downloads/movies` 或者 `/volume1/video:/downloads`

### 3. 操作流程
1. 进入插件详情页，点击 **“🔍 全盘扫描”**。
2. 在下方 **“孤儿种子与辅助清单”** 中确认扫描出的丢失项目（已过滤健康做种项）。
3. 如果未配置 Prowlarr：在底部 **“待处理片名提取区”** 复制列表，自行去 PT 站搜索下载，并在本地对照绝对路径手工辅种。
4. 如果已配置 Prowlarr：点击 **“🧪 灰度测试”**，系统将自动尝试找回前 5 个项目。
5. 去下载器确认：种子是否为暂停或正在校验状态（**若显示 0% 且开始下载新文件，说明路径映射错误，请立即删除任务！**）。
6. 确认一切完美后，点击 **“🚀 全量自动找回”**，泡杯咖啡，让 NAS 在后台慢慢帮你把魔力值赚回来。
7. 遇到突发情况，随时点击 **“🛑 强制停止任务”** 刹车。


## 🛠️ 附录：Prowlarr 极速部署与防封配置指南

如果你还没有部署 Prowlarr（全能索引器），或者不知道如何将其与本插件完美对接，请参考以下精简指南：

#### 1. Docker Compose 一键部署
在你的 NAS 或服务器上创建一个 `docker-compose.yml` 文件，填入以下内容并执行 `docker-compose up -d` 启动：

```yaml
version: '3.3'
services:
  prowlarr:
    image: lscr.io/linuxserver/prowlarr:latest
    container_name: prowlarr
    environment:
      - PUID=1000      # 修改为你的用户UID
      - PGID=1000      # 修改为你的用户组GID
      - TZ=Asia/Shanghai
    volumes:
      - /你的本地路径/prowlarr/config:/config  # 替换为实际存储配置的目录
    ports:
      - 9696:9696
    restart: unless-stopped
```

#### 2. 添加 PT 站点 (以 M-Team 为例)
1. 浏览器访问 `http://你的NAS_IP:9696` 进入 Prowlarr。
2. 首次进入需设置无认证或账号密码登录。
3. 点击左侧菜单 **Indexers** -> 点击顶部 **+ Add Indexer**。
4. 搜索你要添加的站点（如 `M-Team`），推荐在鉴权方式选择 **API Key**（去 M-Team 控制台 -> 安全 -> 实验室获取 API 密钥）而不是 Cookie，稳定性极高。
5. 点击 **Test**，测试通过后点击 **Save**。

#### 3. 🛡️ 极其关键：设置防封限流 (Rate Limit)
*PT 站点（如 M-Team）通常对 API 搜索有严格的 1000次/24小时 限制，若不限制 Prowlarr，几分钟内就会被关小黑屋导致 503 报错！*
1. 在 Prowlarr 顶部菜单点击 **⚙️ Settings**，将顶部的 **Show Advanced**（显示高级设置）切换为开启状态。
2. 回到 **Indexers** 页面，点击你刚添加的 M-Team 右侧的 **扳手图标 (Edit)**。
3. 在弹出的设置窗口往下滚，找到 **Query Limit (查询限制)** 和 **Limit Unit (限制单位)**。
4. 强烈建议设置为：
   - **Query Limit**: `600`
   - **Limit Unit**: `Hours (1)`
   *(这意味着限制每小时最多向站点发起 600 次搜索请求，完美保护你的账号)*
5. **Grab Limit (抓取限制)** 留空即可。点击保存。

#### 4. 对接 SeedRescuer 插件
1. 在 Prowlarr 左侧菜单点击 **Settings** -> **General**。
2. 在 **Security (安全)** 栏目下，找到 **API Key**。
3. 复制这串长长的字符。
4. 回到 MoviePilot -> 插件 -> SeedRescuer 种子找回助手设置页：
   - **Prowlarr 检索源 URL**: 填入 `http://你的NAS_IP:9696`
   - **Prowlarr API Key**: 贴入你刚复制的字符。
5. 保存配置，尽情享受全量自动化找回的快感吧！
