# 种子找回助手 (SeedRescuer)

本插件专为 MoviePilot V2 设计，用于智能扫描本地存储目录，识别丢失种子的影视资源，并自动/手动从 PT 站点找回匹配的种子导入下载器进行自动校验。

## 🌟 核心特性

- **📂 智能树形扫描**：基于影视命名特征（多点分隔符、年份、Sxx等）自动识别发布文件夹。支持多级目录深度钻取，遇到影视文件夹自动截断，不再扫描内部碎片文件。
- **⚖️ 极致匹配算法**：搜索结果与本地文件夹总大小进行比对（精度高达 0.1% 误差），并且会校验本地名称中的核心特征词（如 `iQIYI`, `MWeb`, `WEB-DL`, `REMUX`），确保找回的是同一个版本。
- **🧠 智能标题识别**：优先使用全特征搜索，无结果时调用内置识别链提取“标题+年份”进行兜底搜索。
- **🚀 站点优先级**：搜索结果根据你在配置中点选站点的先后顺序进行排序，优先从你最心仪的站点找回。
- **🔄 路径自动转换**：完美支持 Docker/TR 环境。可将扫描路径映射为下载器路径，确保导入后立即触发自动校验。
- **🛡️ 智能防风控**：内置随机请求延迟机制（可自定义 3~8 秒），有效避免因请求过快被 PT 站点封禁或限流。
- **📊 统计看板与历史**：内置持久化历史记录，已找回的项目永不重复搜索。提供可视化看板，实时展示找回进度。
- **🛠️ 模块化操作**：支持“仅扫描导出清单”、“灰度测试 (前5个)”和“一键全量找回”。

## 📖 使用指南

### 1. 基础配置
- **扫描根路径**：填写你的影视存放路径。支持逗号分隔多个路径，如 `/media/movies, /media/tv`。
- **扫描深度**：如果你的结构是 `根目录/分类/电影名`，建议深度设为 `2` 或 `3`。
- **目标下载器**：选择用于校验的 TR 或 QB。
- **搜索站点**：勾选 PT 站点。注意：**点击选择的先后顺序即为搜索优先级**。
- **随机延迟设置**：配置合理的延迟区间防止触发站点请求频率限制。

### 2. 路径转换 (Docker必填)
如果 MoviePilot 扫描到的路径与下载器看到的路径不一致，请务必配置 `路径转换`。
- 格式：`MoviePilot路径:下载器路径`
- 示例：`/media/movies:/downloads/movies` 或者 `/volume1/video:/downloads`

### 3. 操作流程
1. 进入 **“概览与操作”** 标签页，点击 **“🔍 扫描磁盘”**。
2. 在 **“待找回清单”** 中确认扫描出的项目。
3. 点击 **“🧪 灰度测试 (5项)”** 进行小规模测试。
4. 去下载器确认：种子是否为暂停或正在校验状态（**没有开始下载 0% 的新文件**）。
5. 确认路径无误后，点击 **“🚀 全量全自动找回”** 开始自动化作业。
6. 在 **“统计看板”** 查看最终战果。

## ⚠️ 注意事项
- 本插件默认以 **“暂停”** 状态添加种子。只要路径配置正确，TR 会自动进入 `Verifying (校验中)` 状态。
- 如果测试时发现种子开始重新下载，说明 **路径转换** 设置错误，请立即在下载器中删除任务并调整配置！
